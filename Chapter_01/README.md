 #  1.1预处理指令

含义：***在代码运行之前先被编译器转换为计算机能识别的可执行文件***


## 实验验证

--设计代码demo.cpp：使用#ifdef[id]、eles、#endif来验证预处理指令

    `` #ifdef "A" `` 
    `` const char* str="A";``
    `` else ``
    `` const char* str="B";``
    `` #endif``  

代码逻辑：如果宏A被定义，则str="A";如果没有被定义，则*str="B"*

对比验证：在g++编译时定义宏（-DA)则编译器先预处理#ifdef A，则输出A;反之若没有提前定义宏，则else输出B;

实验结论：代码不做任何改变，只是在编译时加上或者去掉-D，但是结果却截然不同，就说明预处理指令#ifdef发挥作用，在代码运行前就处理了代码。


## 知识点：
-D宏名：在预处理阶段自动定义宏。
