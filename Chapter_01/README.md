 #  1.1预处理指令

含义：***在代码运行之前先被编译器转换为计算机能识别的可执行文件***


## 实验验证

--设计代码使用#ifdef[id]、eles、#endif来验证预处理指令的作用

（ifdef就是if defined的意思）

    `` #ifdef "A"  
     const char* str="A";
     else
     const char* str="B";
     #endif``  

代码逻辑：先编写了一个预处理指令#ifdef A,并定义了一个字符串A，一个字符串B；如果宏被定义，就运行const char* str="A"，如果没有则else,运行const char* str="B"

则编译器运行#ifdef A,

g++时对比验证：

在g++编译时如果定义宏（-DA)则编译器先预处理#ifdef A，则输出A;反之若没有提前定义宏，则else输出B，如下：

>g++ demo.cpp -DA -o demo

>./demo

输出结果：A

>g++ demo.cpp -o demo

>./demo

输出结果：B


实验结论：在两次编译运行的过程中，代码没有做任何改变，只是在编译时加上或者去掉-D来表示是否定义宏，却得到了两个不同的结果，就说明是预处理指令#ifdef发挥作用，在编译器运行前就处理了代码。


## 知识点：

g++中-D宏名：在预处理阶段自动定义宏。

strings 可执行文件 | grep 关键词：将可执行文件中的关键词列出来

strings:把g++编译出来的乱码中的可读字符串文本筛选出来

|管道符：把前一条命令的输出结果直接作为下一条命令的输出

grep:文本检索过滤器，在海量文本中只保留有关键词的行。
