 #  1.1预处理指令

含义：编译器在正式运行源代码之前会优先处理*预处理指令*（包括宏替换，引入头文件等操作），经过预处理净化过后的代码才会最终编译器进行后续的操作，最终生成可执行文件。


## 实验验证

使用#ifdef[id]、eles、#endif条件编译来验证预处理指令的作用

参考可运行代码：

    ``#includ <iosream>
     #ifdef "A"  
     const char* str="A";
     else
     const char* str="B";
     #endif


    int main()
    {cout<<str<<endl;
    return 0``  

代码逻辑：预处理器在预处理阶段检测全局宏A:如果宏A被定义，就只保留const char* str="A"，删除#else分支的全部代码;如果没有宏A没有被定义，则只保留运行const char* str="B"，舍弃上方的全部代码。


对比编译命令：

用g++编译参数-D宏名：在预处理阶段定义全局宏，等价于在代码开头手动书写：#define A

1.编译时定义宏A:

>g++ demo.cpp -DA -o demo

>./demo

输出结果：A

2.编译时不定义任何宏

>g++ demo.cpp -o demo

>./demo

输出结果：B


实验结论：在两次编译运行的过程中，代码没有做任何改变，只是在编译时加上或者去掉-D来表示是否定义宏，却得到了两个不同的结果，就说明是预处理指令#ifdef发挥作用，在编译器运行前就处理了代码。


## 知识点：

g++中-D宏名：在预处理阶段自动定义宏。

strings 可执行文件 | grep 关键词：将可执行文件中的关键词列出来

strings:把g++编译出来的乱码中的可读字符串文本筛选出来

|管道符：把前一条命令的输出结果直接作为下一条命令的输出

grep:文本检索过滤器，在海量文本中只保留有关键词的行。
